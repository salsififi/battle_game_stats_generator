# Battle Game Stats Generator
Un programme pour comprendre enfin, statistiques à l'appui, pourquoi le jeu de bataille est tellement ennuyeux!


## Installation:
- Version de Python nécessaire: 3.10 ou ultérieure.
(Le script utilise les mots-clés 'match' et 'case', apparus avec la version 3.10 de Python).
- Package requis: matplotlib .


## Utilisation:
Lancez le fichier main.py. Vous arriverez alors sur le menu d'accueil, où vous aurez 4 choix:

1. Lire les règles du jeu de bataille
--> il existe de nombreuses versions du jeu de batailles:
cette option vous permettra de savoir laquelle est utilisée dans ce programme.

2. Voir une partie aléatoire entre Alice et Bob
--> la partie durera moins d'une seconde, mais vous pourrez suivre coup par coup les cartes jouées.
Le nombre total de coups joués et le nombre total de "batailles" seront affichés.

3. Générer des statistiques
--> Vous devrez alors préciser le nombre de parties à analyser.
Pour information, si vous lancez 1000 parties, le programme mettra en moyenne 5 secondes à s'exécuter (cette durée peut varier selon la configuration de votre système).
Pour chaque partie, seules les statistiques finales (vainqueur, nombre de coups, nombre de batailles) s'afficheront, puis des statistiques globales sur la série de parties seront générées et présentées sous forme de texte et/ou de graphiques.

4. Quitter le programme
